# Mobile Touch

## 适用范围

Moni 当前目标运行环境是 Android Capacitor，浏览器 F12 移动模式是开发态高保真替身。

## 规则

- 默认按移动端单手触控阅读与操作组织界面。
- 不以桌面 hover 为前提设计关键交互。
- 关键操作热区应避免过小或过密。
- 需要明确区分纵向滚动、横向滑动、长按与拖拽。
- 文本、数字、密码输入应优先使用系统输入法能力，例如 `inputMode="numeric"`、`pattern="[0-9]*"`、`autocomplete` 等。
- 除非产品明确需要自定义键盘，不得在页面内手搓数字键盘替代系统键盘。
- F12 移动模拟下若系统键盘不可见，允许用电脑物理键盘输入；不得因此改变移动端真实交互方案。

## 验收要求

- 关键页面必须在默认移动端视口下验证。
- 若更改视口做专项验证，需在验收记录中写明原因。
