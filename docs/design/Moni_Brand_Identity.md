# Moni — 品牌身份规范

**应用名称**: Moni（AI 原生个人财务助手）
**视觉风格**: Memphis Design
**日期**: 2026-04-07
**文档层级**: Layer 0 — Brand Identity

> 本文档定义 Moni 的品牌身份与视觉资产。具体的色值、字号等数值见 `tailwind.config.js`（Layer 2）；组件视觉语法见 `docs/design/SURFACE_SYSTEM.md`（Layer 1）。

---

## 一、品牌气质

- 风格：轻量 Memphis
- 气质：温暖、活泼、轻松，但不幼稚
- 原则：有情绪、有个性，但金融信息必须清晰可读

---

## 二、品牌色彩语义

### 核心色

| 名称 | 角色 |
|------|------|
| 深底（ink） | Icon 容器、导航栏按钮底色、标题文字、卡片粗边框——Moni 视觉重量的锚点 |
| 米白（surface） | 页面背景、浅色表面——温暖基调的来源 |
| 白（white） | 卡片内底色——内容的纯净底板 |

### 品牌三色装饰（固定组合，所有品牌触点统一使用）

| 元素 | 形状 | 大小关系 |
|------|------|---------|
| 珊瑚圆 | 圆形 | 最大 |
| 天蓝圆 | 圆形 | 中等 |
| 向日黄菱形 | 正方形旋转 18° | 最小 |

三者始终以簇状排列出现，位于品牌 M 的右上方。这是品牌的视觉签名。

### 功能色语义

| 名称 | 用途 |
|------|------|
| 珊瑚红 | 支出金额、危险/警示态 |
| 天蓝 | 信息态、部分分类标签 |
| 向日黄 | 部分分类标签 |
| 薄荷绿 | 收入金额、AI 状态色、健康预算、正向反馈 |
| 琥珀橙 | 预算警戒、未分类提示 |

### 预算状态色语义

| 使用率 | 色调 |
|--------|------|
| < 70% | 薄荷绿系——健康 |
| 70%-90% | 琥珀橙系——警戒 |
| > 90% | 珊瑚红系——超支 |

## 三、字体选型

| 用途 | 字体 | 选型理由 |
|------|------|---------|
| 品牌标题 | Nunito ExtraBold | 圆润饱满，与 Memphis 的活泼气质吻合 |
| 正文/UI | 系统默认无衬线 | 阅读效率优先，不与品牌字体抢注意力 |
| 金额/数据 | Space Mono | 等宽保证对齐，技术感与金融场景匹配 |

## 五、AI 引擎三色流光效果

AI 引擎工作时，两处 UI 元素同步展示品牌三色流光，形成"卡片在处理 ↔ 按钮在运转"的视觉联动。

### 位置 1：正在处理的日卡片边框

- 颜色循环：珊瑚 → 向日黄 → 天蓝 → 薄荷 → 珊瑚
- 3-4 秒一个完整循环，顺时针流动
- 完成后平滑过渡回黑色边框（0.6s）

### 位置 2：导航栏中央按钮

- 按钮外围出现发光轮廓，颜色取品牌三色
- 标签文字变为"运行中"，颜色变薄荷绿
- 引擎停止后光环消散（0.6s），标签恢复"首页"

## 六、Memphis 装饰规范

页面中散布低密度的 Memphis 几何装饰元素，营造活泼氛围但不干扰内容。

- 全局装饰采用程序生成，不手动硬编码摆点
- 每屏 2-4 个装饰元素，位于卡片间隙和页面边角空白处
- 使用品牌色（珊瑚、天蓝、向日黄、薄荷绿），opacity 15%-30%
- 形状：小圆、小三角、短波浪线、小菱形
- 不与文字或数据重叠
- 位置可以随机但每个页面的装饰固定不变（不做动画）
