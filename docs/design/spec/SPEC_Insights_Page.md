# SPEC_Insights_Page

> 洞察页（Insights Page）Layer 3 规格。本页定义 Moni 底部导航 5 tab 结构中位于"设置"右侧、"首页"左侧的一级页面——洞察页的表现层结构、图表组成、交互与动画。
>
> **页面性质**：底部导航的一级页面。
>
> **承担产品职能**：用户主动回看与反思的入口。与首页"当下账本状态"和请教页"AI 主动推送"形成三向时间维度互补——洞察页面向"过去与变化"。

***

## 0. 边界说明

- 本规格仅定义洞察页这一页的表现层。底部导航 5 tab 改造（视觉重排、中间 M 按钮缩放）不在本规格内
- 第三张图"商户 Top N"在本规格内**不实施**，依赖于 `自学习系统改进变更_v1.2.md` §2.2 的传统 NLP 检索方案落地后再单独追加；本规格保留架构兼容但不输出具体设计
- 远期规划的"AI 会话窗口 + 数据检索工具接口"不在本规格内，不预留占位
- 洞察页所有交互均**页内闭环**，不跳转到首页或其他页面（页内闭环原则）

***

## 1. 概念与定位

### 1.1 页面定位

洞察页是用户**主动检索**自身消费数据的入口。它和首页（"当下"）、请教页（"AI 推送"）共同构成洞察 → 当下 → 输入的时间三段。

**关键边界**：

- 洞察页只展示数据、提供视觉关联与交互查看；**不承担数据处理动作**（修改分类、确认审计、删除条目等均不在洞察页发生）
- 洞察页不承载"操作待办"职能（这是请教页的事）
- 洞察页不替代首页流水查看；它给的是"跨时间维度的视图"，首页给的是"当下日期范围内的明细"

### 1.2 数据范围

洞察页所有图表**默认覆盖当前账本的全部交易数据**，**不**受首页 date range 影响。具体每张图各自的时间窗口由该图自身控制：

- 收支柱状图：月模式默认显示最近 12 个月或账本全部范围（取较小者）；周模式默认显示最近 12 周或账本全部范围（取较小者）；用户可在该图上切换粒度
- 分类时间趋势综合表：顶部环状图固定显示**本月**占比；列表条目预算进度条固定为**本月**统计；二级展开图表显示该分类的**全账本月度历史**

### 1.3 页面所属 Chrome

洞察页是底部导航一级页面，**自带 header**、**共享底部导航**。Chrome 归属遵循 `SURFACE_SYSTEM.md` §1.1 中已定义的一级页面规则。

### 1.4 账本切换

洞察页**不**设独立的账本切换器，跟随全局账本状态。用户在其他页切换账本后回到洞察页，洞察页按新账本重新加载所有图表。

## 2. 数据契约

### 2.1 接口返回结构

```typescript
interface InsightsViewData {
  /** 账本元信息 */
  ledger: {
    name: string
    earliestTxDate: string | null   // ISO date, null 表示账本无任何交易
    latestTxDate: string | null
  }

  /** 顶部摘要带数据 */
  summary: {
    totalIncome: number
    totalExpense: number
    /** netCashflow = totalIncome - totalExpense */
    netCashflow: number
    /** 覆盖范围标签的两端 */
    coverageStart: string | null
    coverageEnd: string | null
  }

  /** 收支柱状图数据，按月分组 */
  cashflowByMonth: Array<{
    monthKey: string   // "YYYY-MM"
    income: number
    expense: number
    net: number
  }>

  /** 收支柱状图数据，按周分组 */
  cashflowByWeek: Array<{
    weekKey: string    // "YYYY-Www"
    income: number
    expense: number
    net: number
  }>

  /** 分类时间趋势综合表数据 */
  categoryBreakdown: {
    expense: CategoryBreakdownTabData
    income: CategoryBreakdownTabData
  }
}

interface CategoryBreakdownTabData {
  /** 本月各标签的占比与金额（顶部环状图 + 列表用） */
  currentMonth: Array<{
    /** 标签 ID，'__uncategorized__' 表示未分类 */
    tagId: string
    /** 显示名 */
    tagName: string
    /** 本月该标签总金额 */
    amount: number
    /** 占当月该 direction 总金额的比例 [0, 1] */
    share: number
    /** 该标签本月预算（仅 expense tab；为 null 表示未设预算） */
    budget: number | null
  }>

  /** 各标签的全账本月度历史（二级展开图表用） */
  byTagHistory: Record<string, Array<{
    monthKey: string
    amount: number
  }>>
}
```

### 2.2 tab 数据派生逻辑

`categoryBreakdown.expense` 与 `categoryBreakdown.income` 的数据派生：

```text
1. 先按 transaction.direction 筛选：
   - expense tab → 仅 direction = 'expense' 的条目
   - income tab → 仅 direction = 'income' 的条目

2. 筛选后，统计这批条目实际使用过哪些标签

3. 用这批标签去渲染图表（环状图扇区 / 列表条目 / 二级展开）

后果：同一个标签可能同时出现在两个 tab 里（如果用户用同一标签打过收入和支出条目）；
也可能某个标签只出现在一个 tab 里。
标签是否"属于"哪个 direction 不由用户预先指定，而是由实际使用历史导出。
```

### 2.3 未分类处理

`tagId === '__uncategorized__'` 表示用户既没设 `user_category` 也没设 `ai_category` 的条目。未分类参与所有计算逻辑（占比、列表、二级展开），但视觉上弱化（详见 §7.3）。

## 3. 页面区域结构

洞察页从上到下的固定阅读顺序：

1. **Header 区**（页面标题）
2. **顶部摘要带**（覆盖范围 + 总收支 + 净值）
3. **收支柱状图**（第一张图，月/周可切换）
4. **分类时间趋势综合表**（第二张图，含 tab 切换 / 环状图 / 列表 / 二级展开）
5. **底部导航**（共享 Chrome）

整页可纵向滚动；不使用 tab 横向切分两张大图（垂直堆叠）。

### 3.1 Header 区

| 元素 | 内容 |
|---|---|
| 左侧 | 页面标题"洞察" |
| 右侧 | 无控件（不放 date range 切换器，不放账本切换器） |

Header 采用与首页 / 请教页一致的字体族（`font-brand`）与字号层级。

## 4. 顶部摘要带

### 4.1 视觉骨架

引用 `SURFACE_SYSTEM.md` §2.3 **次级卡（Minor Card）**语法，作为整页第一个内容块。横向布局，单行紧凑。

### 4.2 内容

| 字段 | 显示规则 |
|---|---|
| 覆盖范围标签 | `coverageStart` 至 `coverageEnd`；格式 `YYYY/MM – YYYY/MM`；当账本只有一个月数据时显示单一月份 |
| 总收入 | `summary.totalIncome`；金额前缀 `+`；使用青/薄荷色 |
| 总支出 | `summary.totalExpense`；金额前缀 `−`；使用珊瑚色 |
| 净值 | `summary.netCashflow`；金额带方向前缀（正数 `+`，负数 `−`）；颜色按正负取青/珊瑚 |

四组数据并列展示，金额使用 `font-mono`；标签使用 `font-brand` 较小字号。

### 4.3 空账本降级

`ledger.earliestTxDate === null`（账本无任何交易）时，整张顶部摘要带降级为单行提示"账本暂无数据"，三个金额栏位全部隐藏；同时整页的两张图均替换为空状态（§9）。

## 5. 收支柱状图（第一张图）

### 5.1 视觉骨架

引用 `SURFACE_SYSTEM.md` §2.2 **内容卡（Content Card）**语法。整张卡片作为该图的容器。

### 5.2 卡片头部

| 元素 | 内容 |
|---|---|
| 左侧 | 图标题"收支" |
| 右侧 | 粒度切换控件（月 / 周二选一） |

粒度切换控件引用 `SURFACE_SYSTEM.md` §3.6 **Pill 按钮**语法的二选一并列形式（两个 Pill 紧贴排列，单选互斥）。默认选中"月"。

### 5.3 主体：双柱并列 + 净值线

每个时间点（月或周）渲染：

- **收入柱**：青/薄荷色，竖直；高度对应 `income` 金额
- **支出柱**：珊瑚色，竖直；紧贴收入柱右侧
- **净值线**：折线，连接所有时间点的 `net` 值；正值线在零基线上方，负值在下方；线颜色采用中性深色（不抢柱子主色）

X 轴：时间标签（月模式 `M月`，周模式 `MM/DD` 周首日）
Y 轴：金额刻度，根据数据自动适应；不强制零刻度居中（净值线穿过 Y=0 横轴即可）

### 5.4 时间窗口与横向滚动

| 粒度 | 数据覆盖范围 | 默认视口 | 备注 |
|---|---|---|---|
| 月 | 账本全部有数据的月份 | 最右侧（最近）12 个月或全部（取较小者）可见 | 数据不足 3 个月触发空状态 |
| 周 | 账本全部有数据的周 | 最右侧（最近）12 周或全部（取较小者）可见 | 数据不足 4 周触发空状态 |

**横向滚动查看历史数据**：当账本数据超过默认视口宽度时，图表区域支持**左滑横向滚动**查看更早的时间段；默认停在最右侧（最新数据）；右端对齐、左端自然延伸。

切换粒度时重置到最右侧视口，并平滑过渡（淡出 + 淡入）。

### 5.5 点击交互与焦点规则

**可点击目标**

用户点击以下元素之一时，在该时间点正上方浮现**轻量信息气泡**：

- 收入柱（bar）本体
- 支出柱（bar）本体
- 净值折线上的数据点圆点

**不触发气泡**：柱与柱之间的空白区域、X 轴标签区域、图表背景区域。

**气泡内容与行为**

- 内容：该时间点的 `收入 / 支出 / 净值` 三项数字 + 时间标签
- 气泡视觉：使用 `SURFACE_SYSTEM.md` §六 浮动弹层规则，允许使用阴影
- 气泡随用户点击其他可点击目标而切换；点击图表外区域气泡消失
- **不**触发任何跳转或数据修改

**焦点样式**

图表内所有可点击元素**禁止显示浏览器默认焦点边框**（`outline: none`）；不得在任何交互状态下出现蓝色外框或系统默认 focus ring。

### 5.6 数据不足空状态

```text
月模式：cashflowByMonth.length < 3 → 显示空状态
周模式：cashflowByWeek.length < 4 → 显示空状态
```

空状态文案："数据不足以画出趋势，等账单再积累一些"。
保留卡片骨架，正文区替换为居中提示；不影响其他图。

## 7. 综合表 - 顶部环状图

### 7.1 数据

显示 `categoryBreakdown[activeTab].currentMonth` 的每项 `share` 占比。

仅显示 `amount > 0` 的标签；为零的标签**不参与图表**（不显示扇区，下方列表也不显示对应行）。

### 7.2 中心空心区

环状图中心空心位置显示：

- 主数字：本月该 direction 的总金额（expense tab 显示总支出，income tab 显示总收入）
- 副标签："本月支出" / "本月收入" 短文案

主数字使用 `font-mono`，字号大；副标签 `font-brand`，字号小。

### 7.3 未分类处理

`tagId === '__uncategorized__'` 的扇区：

- 使用**斜杠条纹填充**（视觉上明显区别于纯色扇区）
- 颜色采用中性灰色（不占用任何分类色）
- 扇区位置：始终位于环状图的某固定位置（建议 12 点钟方向起始），便于视觉识别

### 7.4 其他扇区颜色

引用 `SURFACE_SYSTEM.md` §十 **分类色板**为每个标签分配颜色。颜色分配遵循首页"分类概览"的既有逻辑（保持视觉语言一致）。

### 7.5 扇区点击交互

用户点击某扇区：

- 下方分类列表中对应行高亮（背景色加深 / 边框增强 / 缩放轻微，具体由实现层定）
- 列表区如不在可视范围内，平滑滚动至该行
- 扇区本身保持选中态视觉（如轻微缩放或描边）
- **不**触发跳转、不**展开二级**

点击空白区域或当前已选中扇区 → 清除高亮。

## 8. 综合表 - 分类列表

### 8.1 列表项结构

每个有数据的标签渲染一行（按本月金额降序排列；未分类始终排在最后，无论金额大小）。

每行包含：

- **第一行**：
  - 左侧：标签色块（小圆点，颜色与环状图扇区一致；未分类用斜杠条纹小块）+ 标签名
  - 右侧：本月金额（`font-mono`）
- **第二行**：根据 tab 与该标签的预算配置选择以下三种之一（详见 §8.2）
- **展开/折叠图标**：第一行右端，按钮区视觉

### 8.2 第二行的三种形态

**形态 A：预算进度条**（仅 expense tab + 标签 `budget !== null` 时）

- 进度条视觉引用 `SURFACE_SYSTEM.md` 既有进度条规则；如该规则不存在，则需在 surface system 中新增一段"分类预算进度条"规则
- 进度条已用部分宽度 = `amount / budget`
- 文案："已用 ¥X / ¥Y（Z%）"，使用 `font-mono`
- 超支视觉：进度条颜色切换为珊瑚色（warning），文案前缀"已超支"

**形态 B：金额 + 占比双数字**（expense tab + 标签 `budget === null` 时，或 income tab）

- 文案："¥X · 占本月支出 Y%"（expense tab）或 "¥X · 占本月收入 Y%"（income tab）
- 使用 `font-mono` 金额 + `font-brand` 标签文字

**形态 C：未分类的特殊形态**（`tagId === '__uncategorized__'`）

- 同形态 B，但整行视觉权重降低（背景斜杠条纹弱化）

### 8.3 列表项点击交互

点击列表项的**主区域**（除展开按钮外的任意区域）→ 触发该项的展开/折叠：

- 已折叠 → 展开，加载并渲染二级图表（§9）
- 已展开 → 折叠，移除二级图表

**点击展开/折叠按钮**：同上（保留独立按钮作为视觉锚点）。

### 8.4 多个同时展开

允许同时展开多个列表项的二级图表。展开互不影响。

如果同时展开的项过多，列表纵向高度自然增长，整页可滚动；不强制限制最大展开数。

### 8.5 与环状图扇区高亮的联动

列表项除"展开/折叠"状态外，还有"高亮"状态（由环状图扇区点击触发，§7.5）：

- 高亮状态 = 该行被环状图扇区选中导致的视觉强调
- 高亮状态与展开状态**互不冲突**，可同时存在

## 9. 综合表 - 二级展开图表

### 9.1 触发

列表项展开后，在该列表项下方插入二级图表区。

### 9.2 图表类型：柱状图 + 折线图融合

二级图表采用**柱状图与折线图融合**的呈现方式：

- **柱状图**：每个有数据的月份一根柱子；柱高对应该月该标签的绝对金额；柱子宽度与间距由数据点数自适应
- **折线图**：连接所有柱子的顶点；线条颜色与该标签的分类色一致（与环状图扇区、列表项色块同色）
- **柱子颜色**：使用该标签的分类色浅色变体（不抢折线主视觉，作为背景支撑）

二者融合的视觉逻辑：

- 柱状图凸显**每月的绝对高度**，让用户能直接读数
- 折线图凸显**月度间的趋势变化**，让用户看到上升 / 下降的形状
- 12 个月以内的稀疏分布下，两种表达都不互相干扰

### 9.3 最高点标注

二级图表中，**绝对金额最高的那一根柱子**单独标注：

- 该柱顶上方显示金额数字（`font-mono`，较小字号）
- 该柱使用**该标签分类色的深色变体**填充（与其他柱子区分）
- 折线在该点位置以小圆点强调

非最高点的其他柱子不显示金额（避免视觉拥挤）。

### 9.4 X 轴

按月分组；X 轴刻度显示月份标签（`M月` 或 `YY/MM`）。

时间范围 = 该标签在账本中**有数据的月份范围**（不是固定 12 个月）。若该标签全账本只有 3 个月有数据，X 轴只显示 3 个月。

### 9.5 Y 轴

绝对金额。无第二 Y 轴。不显示占比。

刻度自动适应该标签的数据范围；零基线显示。

### 9.6 二级图表的空状态

如果该标签的 `byTagHistory` 数据少于 2 个月（即只有当月数据，无历史趋势）：

- 二级图表替换为提示文案："该分类历史数据不足，再积累一两个月就能看到趋势"
- 不渲染图表骨架

### 9.7 二级图表的视觉骨架

引用 `SURFACE_SYSTEM.md` 内容卡内部的子容器语法（如缩进区块、内嵌区）。二级图表是列表项的下挂内容，不是独立卡片。

## 10. 整页空状态

| 状态 | 触发条件 | 视觉处理 |
|---|---|---|

## 11. 动画时序

### 11.1 粒度切换（柱状图月/周切换、综合表收支 tab 切换）

```text
1. 用户点击新档位
2. 阶段 A：当前图表整体淡出
   - duration：~180ms
3. 阶段 B：新数据重算并渲染
4. 阶段 C：新图表整体淡入
   - duration：~220ms
5. 总时长：~400ms
```

### 11.2 柱状图柱子点击 → 气泡浮现

```text
气泡从柱顶向上滑入 + 淡入
- 入场偏移：~8px
- duration：~200ms
- easing：ease-out
```

气泡切换：先淡出旧气泡（~120ms），再淡入新气泡（~180ms）；不交叠。

### 11.3 环状图扇区点击 → 列表项高亮

```text
列表项背景色与缩放过渡
- duration：~240ms
- easing：ease-in-out
列表区滚动至高亮项（如需）
- 平滑滚动 ~400ms
```

### 11.4 列表项展开 → 二级图表入场

```text
1. 列表项之间高度撑开（容纳二级图表所需高度）
   - duration：~240ms
2. 二级图表内容淡入（柱子从底部生长 + 折线从左到右描画）
   - 柱子生长：~360ms，ease-out
   - 折线描画：~480ms，与柱子并行启动，略晚结束
3. 最高点圆点与金额标注最后淡入
   - duration：~200ms，延迟 ~300ms 启动
```

### 11.5 列表项折叠 → 二级图表退场

```text
反向折叠
- 内容淡出：~180ms
- 列表项高度收缩：~220ms
```

### 11.6 数据加载

整页首次加载或账本切换后：

- 各内容区使用骨架占位（skeleton），由上至下依次填充实际数据
- 整体加载体感目标：< 500ms 完成首屏可见内容渲染

## 12. 与其他模块的接口约定

### 12.1 数据消费

洞察页通过 `LedgerService`（或同等口径的读模型层）一次性获取整页所需的全部聚合数据：

```typescript
/** 获取当前账本的洞察页视图数据 */
function getInsightsViewData(): Promise<InsightsViewData>
```

实现层负责对 transaction 数据进行聚合（按月分组、按周分组、按 direction 分桶、按标签分桶）。聚合应在数据层完成，不在表现层做循环统计。

### 12.2 账本切换响应

订阅全局账本变化事件；切换发生时重新调用 `getInsightsViewData` 并替换整页数据。

### 12.3 不消费的事项

洞察页**不**调用以下接口：

- 任何写入接口（`verifyTransaction` / `updateCategory` 等均不存在于本页）
- 请教页相关接口（请教队列、AI 不确定状态等）
- AI 引擎控制接口
