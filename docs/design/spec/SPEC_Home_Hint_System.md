# 首页情景提示系统专项规格

## 1. 文档定位

本文档单独定义首页“看板与统计区之间”的情景提示系统。

它只负责中部情景提示卡，不负责顶部轻提示。

顶部轻提示与中部情景提示卡必须分工明确：

1. 顶部轻提示：承接即时、过程态、轻播报反馈
2. 中部情景提示卡：承接首页下一步动作引导，尤其是新用户上手

本轮优先目标固定为：

```text
在不做首次启动引导、且没有任何测试用户交易数据的情况下，
让新用户直接进入空首页后，仍能靠情景提示卡逐步完成上手。
```

补充说明：

- `secure_config` 不属于“用户交易数据”，不影响本规格中的“空首页”判断
- 本规格优先级高于旧的泛化提示池设想

---

## 2. 展示位置与基础行为

### 2.1 展示位置

情景提示卡固定出现在：

```text
顶部看板下方
统计摘要栏上方
```

即首页主舞台中的：

```text
DisplayBoard
↓
Hint Card
↓
StatsBar
```

### 2.2 基础行为

1. 有内容时显示，无内容时整块消失
2. 同一时刻只展示一张主提示卡
3. 不是弹窗，不打断页面阅读
4. 允许用户主动关闭
5. 关闭只影响当前会话，不直接视为“该引导步骤已完成”

---

## 3. 与顶部轻提示的边界

以下内容固定不进入中部情景提示卡：

1. “有尚未学习的实例，AI 正在自动学习”
2. “已学习完成，开始进行分类”
3. 其他纯过程态播报

这些都属于顶部轻提示职责。

中部情景提示卡只承接：

1. 新用户下一步应该做什么
2. 某个阶段完成后，下一步该去哪里
3. 必要时可直接跳转或触发动作的引导

---

## 4. 卡片结构规格

### 4.1 固定结构

卡片采用三段式结构：

1. 左侧信息区
2. 右侧动作区（可选）
3. 最右侧关闭区

### 4.2 左侧信息区

左侧信息区固定为两行：

1. 上方标题
2. 下方小字描述

语义要求：

- 标题负责说清“现在该做什么”
- 描述负责解释“为什么要做”或“怎么做”

### 4.3 右侧动作区

右侧动作区是可选区域，不是所有卡片都必须有。

使用规则：

1. 若该步骤存在明确、单一、可立即执行的页面跳转或动作，则显示快捷操作按钮
2. 若该步骤本质上是说明型提示，而不是按钮可替代的单一动作，则不显示快捷操作按钮

按钮要求：

1. 文案必须短，例如“去设置”“去导入”“去分类”
2. 点击后必须触发真实页面跳转或真实动作
3. 禁止保留无效占位按钮

### 4.4 最右侧关闭区

关闭区固定为：

```text
×
```

规则：

1. 所有卡片都允许关闭
2. 关闭仅表示“本次会话先隐藏”
3. 下次进入软件时，系统必须重新根据业务事实判断是否需要再次出现

### 4.5 视觉延续

视觉继续沿用首页现有 Hint Card 语法：

1. 暖色底
2. 暖色描边
3. 小圆角
4. 无阴影主导

若未来视觉系统升级，以 `SURFACE_SYSTEM.md` 为上位约束；本文不重复定义 token 明细。

## 5. 新用户上手主线

### 5.1 主线目标

当用户跳过首次启动引导、直接进入空首页时，系统应按下面的顺序，只展示“第一张尚未完成”的提示卡：

1. 设置自述
2. 设置月预算
3. 导入账单
4. 开启 AI 分类
5. 学习分类后的交互方式

### 5.2 顺序原则

这五步不是并列提示池，而是一条顺序链路。

因此：

1. 前一步未完成时，后一步不应抢先出现
2. 某一步被关闭但未完成，只在本次会话隐藏
3. 下次进入软件时，重新从头检查，继续显示第一张尚未完成的卡片

## 6. 卡片清单总表

| 顺序 | 卡片 ID | 标题方向 | 描述方向 | 是否显示快捷操作按钮 | 快捷操作 | 触发条件 | 永久退出条件 |
|------|------|------|------|------|------|------|------|
| 1 | `onboarding_set_self_description` | 先设置自述 | 让 AI 更理解用户生活语境 | 是 | 跳转到自述设置页，按钮文案 `去设置` | 当前用户仍处于默认 demo 自述状态，且尚未完成该步骤 | 用户已把默认 demo 自述改成自己的真实内容并保存 |
| 2 | `onboarding_set_monthly_budget` | 先设置月预算 | 帮助首页尽快形成预算感知与节奏反馈 | 是 | 跳转到预算设置页，按钮文案 `去设置` | 自述已完成；当前账本尚无有效月预算；且尚未完成该步骤 | 当前账本已存在有效月预算 |
| 3 | `onboarding_import_bill` | 先导入一份账单 | 没有交易数据时无法进入真实使用状态 | 是 | 跳转到记账页导入入口，按钮文案 `去导入` | 前两步中已到达该步骤；当前账本还没有首份导入账单 | 当前账本已存在至少一条通过账单导入产生的交易 |
| 4 | `onboarding_start_ai_classification` | 试着开启 AI 分类 | 教用户长按底部 AI 按钮启动分类 | 否 | 无 | 用户已完成首次导入，但还没有“曾经主动启动过 AI 分类”的事实 | 用户至少成功发起过一次 AI 分类启动 |
| 5 | `onboarding_learn_post_ai_interaction` | 学会和结果交互 | 教用户长按拖拽改分类，或单击进入详情页 | 否 | 无 | 用户已经启动过至少一次 AI 分类，但还未完成分类后交互学习 | 用户至少完成过一次”长按拖拽分类”或”单击进入详情页查看/修正” |
| 6 | `import_reminder` | 好久没导入账单了 | 距上次导入天数/周数/月数（动态文案，见下方规则） | 是 | 跳转到记账页导入入口，按钮文案 `去导入` | onboarding 主线全部完成；用户曾有过至少一次导入记录（`lastBillImportAt` 不为 null）；距上次导入超过 3 天 | 本次会话关闭后隐藏；下次重进重算距离天数 |
| 7 | `budget_warning` | 本月预算已使用较多 | 提醒用户最近几天控制节奏 | 否 | 无 | onboarding 主线已无更高优先级卡片；总预算状态从 `healthy` 跳到 `warning` | 本次会话关闭后隐藏；状态解除后自然消失 |
| 8 | `budget_exceeded` | 本月预算已超出 | 告知用户总预算已越界 | 否 | 无 | onboarding 主线已无更高优先级卡片；总预算状态从 `warning` 跳到 `exceeded` | 本次会话关闭后隐藏；状态解除后自然消失 |
| 9 | `budget_setup_nudge` | 要不要设一个月预算 | 当已有一定流水但还没预算时，提示用户补设预算 | 是 | 跳转到预算设置页，按钮文案 `去设置` | onboarding 主线已无更高优先级卡片；已有足够流水；当前账本仍无有效月预算 | 用户已设置有效月预算 |
| 10 | `category_budget_exceeded` | 某些类别已经超支 | 告知哪些分类预算已经越界 | 否 | 无 | onboarding 主线已无更高优先级卡片；分类预算开启且存在超支分类 | 本次会话关闭后隐藏；超支状态解除后自然消失 |
| 11 | `category_budget_invalidated` | 分类预算已失效 | 标签结构变化后提醒重新配置分类预算 | 是 | 跳转到预算设置页，按钮文案 `去设置` | onboarding 主线已无更高优先级卡片；分类预算因标签结构变化失效 | 用户重新保存分类预算配置 |

**`import_reminder` 动态文案规则：**

| 距上次导入天数 | 描述文案 |
|---|---|
| 4–7 天 | 距上次导入已 X 天，趁早把账单补上。 |
| 8–30 天 | X 周没有导入了，要补一下吗？（X = floor(days/7)，最小 1） |
| 31 天以上 | X 个月没有导入了，数据会有空缺。（X = floor(days/30)，最小 1） |

注意：距上次导入 ≤ 3 天时不触发本卡片。

表格说明：

1. “是否显示快捷操作按钮”必须按实际场景判断，不是所有卡片都强制有按钮
2. 第 4、5 张卡属于说明型卡片，明确不带快捷操作按钮
3. 第 6 张导入提醒卡优先级高于预算类提示，但低于 onboarding 主线
4. 现有预算类提示卡已经并入同一总表，但优先级低于 onboarding 主线和导入提醒
5. 若后续产品确认某张说明型卡片也需要真实动作按钮，应另行更新本表，不得在实现里私自扩展

## 7. 完成态判断规则

### 7.1 设置自述

完成条件：

```text
用户已修改并保存默认 demo 自述，而不是继续保持系统提供的默认 demo 文案
```

禁止把“非空”直接视为完成。
因为当前系统存在默认 demo 自述，只有当用户明确改成自己的内容后，才能算完成该步骤。

### 7.2 设置月预算

完成条件：

```text
当前账本已存在有效月预算
```

这里特指总预算。
分类预算不属于 onboarding 主线完成条件，而属于后续可出现的扩展情景提示能力。

### 7.3 导入账单

完成条件：

```text
当前账本至少存在一条通过账单导入链路产生的交易
```

手动记账不等价于“已完成导入账单”。

### 7.4 开启 AI 分类

完成条件：

```text
用户曾经主动发起过至少一次 AI 分类启动
```

这里要求是“真实启动行为”而不是只看按钮曝光。

### 7.5 学习分类后交互方式

完成条件：

```text
用户至少完成过以下任一真实交互：
1. 长按拖拽改分类
2. 单击交易进入详情页查看或修正
```

---

## 8. 重进软件后的重算规则

每次重新进入软件时，系统必须重新执行一次完整判断：

1. 读取五个 onboarding 步骤的真实完成态
2. 从顺序 1 开始查找第一张尚未完成的卡片
3. 若存在，则显示该卡片
4. 若五步都完成，再进入非 onboarding 的现有提示卡候选选择
5. 若五步与现有提示卡都没有可展示项，则中部情景提示卡整体消失

注意：

1. “本次会话里被关闭过”不应写成永久完成态
2. “已经完成”才允许永久退出

## 9. 推荐读模型结构

本轮推荐逻辑层输出显式结构，而不是让表现层拼判断：

```json
{
  "hintCard": {
    "id": "onboarding_set_self_description",
    "type": "onboarding_step",
    "title": "先告诉 Moni 你是谁",
    "description": "写一段自述，AI 后续分类会更贴近你的真实生活语境",
    "dismissible": true,
    "action": {
      "enabled": true,
      "label": "去设置",
      "kind": "navigate",
      "target": "settings_self_description"
    }
  }
}
```

若某卡片不带快捷操作按钮，则：

```json
{
  "action": {
    "enabled": false
  }
}
```

## 10. 当前实现状态

截至 0.4.3，已实现项：

1. ✅ onboarding 主线五步（自述、月预算、首次导入、开启 AI、分类后交互）全部落地
2. ✅ 首页只消费 `hintCards[0]`，`HomeHintSystemBuilder` 输出顺序即优先级
3. ✅ 关闭行为为本次会话隐藏；重进后按完成态重算
4. ✅ 快捷操作按钮已接通真实跳转（自述设置页、预算设置页、记账页导入入口）
5. ✅ `HomeHintStateManager` 持久化 `hasStartedAiProcessing` / `hasCompletedPostAiInteraction` / `lastBillImportAt`
6. ✅ 导入提醒卡（`import_reminder`）已实现，动态文案按天/周/月显示

残留差距：

- 预算类提示卡（`budget_warning` / `budget_exceeded` / `category_budget_exceeded`）目前无快捷操作按钮，如后续产品需要可按表格说明补充
