# SPEC_DragDetailPanel_and_TransactionDetailPage.md 增补
## 纠错弹窗（ReasonDialog）外侧蒙版语义统一

> 本文档是对 `SPEC_DragDetailPanel_and_TransactionDetailPage.md` 的增补，统一所有触发 `ReasonDialog`（分类纠错时的可选理由输入弹窗）的路径上，**外侧蒙版的行为语义**。增补内容可直接合入原文档末尾，或作为独立约束文件被各路径主规格引用。

***

## 1. 背景

`ReasonDialog`（分类纠错时的可选理由输入弹窗）是 Moni 中分类纠错流程的下游产物。原规格 `SPEC_DragDetailPanel_and_TransactionDetailPage.md` 仅引用其"既有口径"，未对外侧蒙版行为做明确约束。

`ReasonDialog` 当前由两个路径触发：

1. **首页拖拽路径**：用户在首页拖拽某条目到分类格上完成投放后，弹出 `ReasonDialog`
2. **请教页拖拽路径**（v1.2 后新增）：用户在请教页对某可操作条目执行拖拽纠错后，弹出同款 `ReasonDialog`

未来若新增其他分类纠错入口（如"批量纠错时的统一理由"流程），仍复用 `ReasonDialog`。

原弹窗的两个按钮：

- **跳过**：应用纠错（分类变更生效），但不填写 `user_note`
- **完成**：应用纠错（分类变更生效），并以输入框内容填写 `user_note`

原**外侧蒙版（点击 ReasonDialog 卡片外的暗化区域）**行为为 **无作用**。

随请教页规格冻结，新增"取消纠错"需求——用户可能因手滑导致拖拽完成、但希望放弃本次纠错。

***

## 2. 增补：外侧蒙版语义

### 2.1 语义统一

`ReasonDialog` 的外侧蒙版语义从"无作用"统一调整为 **"取消纠错"**：

| 触发位置 | 外侧蒙版语义 | 状态回滚结果 |
|---|---|---|
| 首页拖拽路径 | 点击外侧蒙版 → 取消纠错 | 本次分类不被应用，条目保持拖拽前状态 |
| 请教页拖拽路径 | 点击外侧蒙版 → 取消纠错 | 本次分类不被应用，条目仍留在请教页视图中 |
| 未来新增路径 | 同上 | 同上 |

### 2.2 与"跳过" / "完成"的三态差异

`ReasonDialog` 共有三种退出路径，语义必须清晰互斥：

| 动作 | 是否应用分类变更 | 是否写入 user_note | 触发方式 |
|---|---|---|---|
| **完成** | ✓ 是 | ✓ 写入输入框内容 | 点击"完成"按钮 |
| **跳过** | ✓ 是 | ✗ 不写入 | 点击"跳过"按钮 |
| **取消** | ✗ 否（分类回滚） | ✗ 不写入 | 点击外侧蒙版 |

三个动作在实现层必须有独立的状态分支，不可借用"跳过 + 清空输入"等模拟方式表达"取消"。

## 3. 弹窗卡片增加提示文案

### 3.1 文案

`ReasonDialog` 卡片内增加一行轻量提示文字：

```
手滑了？点击卡片外侧空白处取消修正
```

### 3.2 视觉规则

- **字号**：小于主标题、主输入框、主按钮文案；与卡片内的次级说明文字层级一致
- **颜色**：使用辅助色（如 60–70% 不透明度的中性灰），不抢主操作按钮（"跳过" / "完成"）的视觉权重
- **位置**：建议位于卡片底部（按钮区下方）或顶部（主标题之下）；具体位置不破坏现有弹窗主体内容布局；不允许放置在输入框 placeholder 中
- **独立行**：必须独立呈现一行，不与其他文案合并

### 3.3 提示文案出现条件

提示文案在弹窗打开后始终可见，不随用户输入隐藏；不需要根据触发路径变化文案内容。

***

## 4. 外侧蒙版触发判定

### 4.1 判定边界

外侧蒙版点击事件的"取消"判定边界：

- 点击事件的 target **不属于** `ReasonDialog` 卡片本身或其子元素
- 即：指点事件发生在弹窗 portal 根容器的蒙版区域（卡片以外的暗化区域）

### 4.2 实现建议

可在弹窗 portal 根容器上绑定 `onClick` → 触发取消逻辑；在内部卡片容器上 `stopPropagation`，避免卡片内点击误触发取消。

### 4.3 拖拽手势的特殊处理

由于 `ReasonDialog` 的两个调用路径均涉及拖拽手势：

- **首页拖拽**：用户的拖拽手指 `pointerup` 即触发弹窗打开。打开瞬间用户可能尚未抬起手指、或刚抬起；表现层应保证弹窗打开后的首个 `pointermove` / `pointerdown` 才被视为有效交互，避免拖拽尾帧的"惯性触摸"被误判为外侧蒙版点击
- **请教页拖拽**：同上

具体实现可采用打开后的 100ms `cooldown` 窗口，或要求外侧点击必须是完整的 `pointerdown → pointerup` 序列。

## 5. 各触发路径下的状态回滚行为

### 5.1 首页拖拽路径

外侧蒙版触发"取消"后：

- **不调用** `LedgerService` 的分类写入接口
- 拖拽前的条目状态完全保留（`user_category` / `is_verified` / `user_note` / `ai_category` 等所有字段均不变）
- 拖拽蒙版（DragDetailPanel + 分类格）正常退场动画
- 视觉层：条目回到首页流水列表原位，无任何视觉变化

### 5.2 请教页拖拽路径

外侧蒙版触发"取消"后：

- 不调用任何写入接口
- 条目仍属于请教页可视集合
- 若条目原本是可操作条目（可拖拽 / 可左滑），继续保持可操作状态
- **不**播放左滑退场动画
- **不**触发会话快照更新（参见 `自学习系统改进变更_v1.2.md` §2.3-E）
- 视觉层：条目回到请教页所在日期卡片中原位置

### 5.3 通用约束

无论哪种路径，"取消"动作的核心不变量：

> **取消 = 整个纠错事件视为从未发生；所有副作用（数据写入、视觉位移、快照更新）必须完整回滚。**

***

## 6. 不影响的部分

本增补仅修改外侧蒙版语义与提示文案；以下既有规格保持不变：

- `ReasonDialog` 的主体布局、字段、输入框规则
- "跳过" / "完成"按钮的位置、样式、动作
- `ReasonDialog` 的视觉风格（按钮形态、卡片圆角、字体等）由 `SURFACE_SYSTEM.md` 与各路径主规格继承
- `ReasonDialog` 的打开 / 关闭动画
